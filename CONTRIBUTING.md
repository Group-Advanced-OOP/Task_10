# Contribution Guide

## Branch Rules
- Each member must create their own branch.
- Branch names should include your name or task (example: `senior-ui`, `logic-task`, `testing-branch`).

## Commit Rules
- Every member must make 2 commits on their branch.
- Commit messages should clearly describe what was done.

## Pull Requests
- Before merging into `main`, members must:
  - Create a Pull Request (PR)
  - Add a short description of what they changed
  - Wait for approval before merging

## Issues
- All tasks must be created as Issues.
- Issues must have a title, description, and be linked to the Project Board.

## File Uploads
- Images and screenshots should be uploaded to the `/Screenshots` folder.
- Documentation files go into the `/Documentation` folder.
